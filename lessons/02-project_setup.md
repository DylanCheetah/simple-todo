# Lesson 02: Project Setup

Now that we have installed the development tools we need, we can begin setting up our project. First we need to create a folder for our project. Create a folder called `simple-todo`. Next we need to create a virtual environment to install our project dependencies into. Open a terminal inside your project folder and execute the following command:
```sh
python -m venv venv
```

Then you need to activate your virtual environment:
* Windows: execute `cmd /k venv\Scripts\activate`
* Linux: execute `source venv/bin/activate`
* MacOS: execute `source venv/bin/activate`

Next, execute the following command to open your project folder in Visual Studio Code:
```sh
code .
```

Now we need to create a `requirements.txt` file which lists the Python packages needed by our project. For now we will just add Django and django-environ:
```
Django
django-environ
```

Next we need to install our project dependencies. Click Terminal > New Terminal to open a new terminal in Visual Studio Code and execute the following command to install the project dependencies:
```sh
pip install -r requirements.txt
```

Now we need to setup a new Django project. Django project names must be valid Python package names and must not conflict with the names of existing packages. Execute the following command to create a Django project called "simple_todo":
```sh
django-admin startproject simple_todo
```

At this point your project structure should look like this:
```
simple-todo/
    simple_todo/
        simple_todo/
            __init__.py
            asgi.py
            settings.py
            urls.py
            wsgi.py
        manage.py
    venv/
    requirements.txt
```

Next we need to setup django-environ. django-environ provides a convenient way to manage one or more project environments. It's common to have both development and production environments for a project. Let's create a template for our environment files. Create `simple-todo/simple_todo/.env.dist` with the following content:
```env
# Debug state (on for debug; off for production (default))
DEBUG=on

# Secret key (generate one with django.core.management.utils.get_random_secret_key)
SECRET_KEY=key

# Database URL
DATABASE_URL=sqlite:///db.sqlite
```

Now we can create our development environment file. Copy `simple-todo/simple_todo/.env.dist` and name it `.env`. If you're using source control, make sure this file is excluded from source control. To generate a secret key for your development environment, execute the following command:
```sh
python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"
```

Then copy your secret key into your `.env` file. Your project structure should look like this now:
```
simple-todo/
    simple_todo/
        simple_todo/
            __init__.py
            asgi.py
            settings.py
            urls.py
            wsgi.py
        .env
        .env.dist
        manage.py
    venv/
    requirements.txt
```
